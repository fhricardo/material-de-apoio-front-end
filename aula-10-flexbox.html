Flexbox Layout - Introdução

O Módulo Flexbox Layout (ou Flexible Box), que se tornou uma Candidate Recommendation da W3C em outubro de 2017, foi criado para oferecer uma maneira mais eficiente de organizar, alinhar e distribuir espaço entre os itens de um container — mesmo quando seus tamanhos são desconhecidos ou dinâmicos (daí o termo "flex", ou flexível).

Princípio Básico

A ideia principal do Flexbox é permitir que o container altere a largura, altura e ordem dos seus itens para melhor preencher o espaço disponível. Isso é especialmente útil para se adaptar a diferentes tipos de telas e dispositivos.

  • Se houver espaço livre, os itens se expandem para ocupá-lo.
  • Se o espaço for insuficiente, os itens se encolhem para evitar vazamentos (overflow).

Flexbox vs. Layouts Tradicionais

Diferente dos modelos tradicionais — como o block (baseado em disposição vertical) e o inline (baseado em disposição horizontal) —, o Flexbox não depende de direção fixa. Isso o torna ideal para:

  • Ajustes em orientações diferentes (como rotação de tela).
  • Redimensionamento dinâmico (stretching e shrinking).
  • Aplicações complexas onde os layouts comuns não são suficientes.

Quando Usar?

O Flexbox é mais indicado para:

  • Componentes de aplicações.
  • Pequenos e médios layouts.

Para layouts em grande escala, recomenda-se o uso do CSS Grid Layout, que oferece um controle mais robusto sobre estruturas complexas.

Conceitos Básicos e Terminologia

Como o flexbox é um módulo completo e não uma única propriedade, ele envolve diversos conceitos e um conjunto específico de propriedades. Algumas dessas propriedades são aplicadas ao container (elemento pai, chamado de "flex container"), enquanto outras são aplicadas aos filhos (chamados de "flex items").

Terminologia do Flexbox

Eixo Principal (Main Axis)

O eixo principal de um flex container é o eixo primário ao longo do qual os flex items são dispostos. Atenção: ele não é necessariamente horizontal; isso depende da propriedade flex-direction (ver abaixo).

  • main-start | main-end - Os flex items são posicionados dentro do container a partir de main-start em direção a main-end.
  • main size - A largura ou altura de um flex item (dependendo da dimensão principal) é chamada de main size. A propriedade de main size será 'width' ou 'height', conforme a dimensão principal.

Eixo Transversal (Cross Axis)

O eixo perpendicular ao eixo principal é chamado de eixo transversal. Sua direção depende da direção do eixo principal.

  • cross-start | cross-end - As linhas flex são preenchidas com itens e posicionadas no container a partir de cross-start em direção a cross-end.
  • cross size - A largura ou altura de um flex item (dependendo da dimensão transversal) é chamada de cross size. A propriedade de cross size será 'width' ou 'height', conforme a dimensão transversal.
