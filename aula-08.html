CSS - Aula 08

Display, Visibility e Overflow

Introdução

Nesta aula, vamos conhecer algumas propriedades fundamentais para o controle visual dos elementos na tela: display, visibility e overflow. Essas propriedades afetam diretamente como os elementos aparecem e se comportam dentro do layout.

1. Propriedade display

Controla como um elemento é exibido na tela. Os valores mais comuns são:

  • block: Ocupa toda a linha. Ex: <div>, <section>
  • inline: Fica na mesma linha. Ex: <span>, <a>
  • inline-block: Mistura dos dois: respeita largura/altura, mas fica na mesma linha
  • none: Oculta completamente o elemento (não ocupa espaço)
p {
          display: inline;
        }
        
        span {
          display: block;
        }
        
        div.oculto {
          display: none;
        }

2. Propriedade visibility

Define se o elemento está visível ou não, mas diferente do display: none, ele ainda ocupa espaço.

  • visible: o padrão, elemento visível
  • hidden: o elemento fica invisível, mas continua ocupando espaço no layout
h1 {
          visibility: hidden;
        }

3. Propriedade overflow

Controla o que acontece quando o conteúdo ultrapassa o tamanho do container.

  • visible: mostra tudo, mesmo fora do container (padrão)
  • hidden: corta o conteúdo que ultrapassar
  • scroll: adiciona rolagem sempre
  • auto: adiciona rolagem só se necessário
.caixa {
          width: 200px;
          height: 100px;
          overflow: auto;
        }

Comparando display: none vs visibility: hidden

Propriedade Elemento é visível? Ocupa espaço?
display: none Não Não
visibility: hidden Não Sim

Boas práticas

  1. Use display: none quando quiser remover totalmente um elemento.
  2. Use visibility: hidden se precisar manter o layout intacto.
  3. Use overflow: auto para garantir que conteúdo não ultrapasse áreas fixas.

Conclusão

Essas propriedades são essenciais para ter controle visual dos elementos. Entender como cada uma delas afeta o layout te ajuda a construir páginas organizadas, com elementos bem comportados na tela. A próxima etapa será mergulhar no Flexbox, que vai abrir um mundo de possibilidades para organizar esses elementos com flexibilidade e responsividade.
