HTML - Aula 04

Tags DIV e Elementos Semânticos

Introdução

Nesta aula, vamos explorar duas abordagens fundamentais para estruturar conteúdo em HTML: a tag <div> (genérica) e as tags semânticas introduzidas no HTML5.

A Tag DIV

A tag <div> é um container genérico que agrupa elementos para fins de estilo ou organização.

Características:

  • É um elemento em bloco (ocupa toda a largura disponível)
  • Não tem significado semântico
  • Muito utilizada antes do HTML5 para estruturar layouts

Exemplo Básico:

<div>
    <h2>Título da Seção</h2>
    <p>Conteúdo da seção...</p>
</div>

Elementos Semânticos do HTML5

Com o HTML5, foram introduzidas tags que dão significado às diferentes partes de uma página web:

<header>
Representa o cabeçalho da página ou de uma seção. Normalmente contém logotipos, menus e elementos introdutórios.
<nav>
Define uma seção de navegação com links para outras páginas ou partes da mesma página.
<main>
Especifica o conteúdo principal e único da página. Deve haver apenas um por documento.
<article>
Representa conteúdo independente que pode ser distribuído separadamente, como posts, notícias, etc.
<section>
Define uma seção genérica de conteúdo, geralmente com um título.
<aside>
Representa conteúdo relacionado mas não essencial, como barras laterais ou caixas de informação.
<footer>
Define o rodapé da página ou de uma seção. Normalmente contém informações de autoria, links e direitos.
<figure> e <figcaption>
Usados para agrupar imagens com suas legendas.

Comparação: DIV vs Semântica

Característica DIV Elementos Semânticos
Significado Nenhum Específico
SEO Neutro Melhorado
Acessibilidade Neutro Melhorada
Leitura do código Menos clara Mais clara

Exemplo de Estrutura Semântica

<!DOCTYPE html>
<html>
<head>
    <title>Página Semântica</title>
</head>
<body>
    <header>
        <h1>Meu Site</h1>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#sobre">Sobre</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <article>
            <h2>Título do Artigo</h2>
            <p>Conteúdo do artigo...</p>
        </article>
        
        <aside>
            <h3>Leia também</h3>
            <p>Conteúdo relacionado...</p>
        </aside>
    </main>
    
    <footer>
        <p>© 2025 Meu Site</p>
    </footer>
</body>
</html>

Quando Usar DIV

Apesar dos elementos semânticos, a DIV ainda tem suas utilidades:

  • Para agrupar elementos sem significado semântico específico
  • Como container para estilização CSS
  • Quando nenhum elemento semântico se encaixa no propósito

Boas Práticas

  1. Prefira elementos semânticos quando possível
  2. Use DIVs apenas para fins de estilização ou agrupamento genérico
  3. Não aninhe elementos semânticos sem necessidade
  4. Mantenha uma hierarquia lógica
  5. Use ARIA roles quando necessário para melhorar acessibilidade

Exercício Prático

Converta a seguinte estrutura baseada em DIVs para uma estrutura semântica:

<div id="cabecalho">...</div>
<div id="menu">...</div>
<div id="conteudo">
    <div class="post">...</div>
    <div class="post">...</div>
</div>
<div id="rodape">...</div>

Conclusão

Os elementos semânticos do HTML5 trazem significado à estrutura da página, melhorando acessibilidade, SEO e manutenção do código. Enquanto as DIVs continuam úteis para agrupamentos genéricos, o uso adequado da semântica é uma prática essencial no desenvolvimento web moderno.
