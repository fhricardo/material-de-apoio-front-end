Box Model

O que é o Box Model?

O Box Model (ou Modelo de Caixa) é o modo como o CSS enxerga cada elemento HTML. Cada elemento é tratado como uma “caixa” que pode conter outras caixas, com propriedades que definem seu tamanho e espaçamento.

Partes de um Box

  • Content: o conteúdo em si (texto, imagem, etc.)
  • Padding: espaço interno entre o conteúdo e a borda
  • Border: contorno do elemento
  • Margin: espaço externo que separa o elemento de outros

Ilustração do Modelo de Caixa

Propriedades CSS do Box Model

1. width e height

Definem a largura e altura do conteúdo (não incluem padding, border ou margin).

div {
          width: 100px;
          height: 100px;
        }

2. padding

Espaçamento interno. Afasta o conteúdo da borda.

/* Um valor para todos os lados */
div {
    padding: 20px;
}

/* Dois valores: vertical | horizontal */
div {
    padding: 20px 30px;
}

/* Quatro valores: top | right | bottom | left */
div {
    padding: 20px 30px 10px 15px;
}

3. border

Define espessura, estilo e cor da borda do elemento.

div {
    border: 2px solid black;
}

4. margin

Espaço externo. Afasta o elemento dos outros.

div {
    margin: 20px;               /* todos os lados */
    margin: 10px 15px;          /* vertical | horizontal */
    margin: 10px 15px 20px 25px;/* top | right | bottom | left */
}

box-sizing

Essa propriedade muda como a largura e altura são calculadas. O padrão é content-box, onde padding e border são somados à largura e altura.

Com box-sizing: border-box, o padding e a border são incluídos dentro da largura e altura definidas, facilitando o controle do layout. 

div {
    width: 100px;
    height: 100px;
    padding: 20px;
    border: 2px solid black;
    box-sizing: border-box;
}

Resumo Visual

  • content-box: largura + padding + border = tamanho final
  • border-box: largura = tamanho final (o padding e a borda se ajustam dentro)

Boas práticas

  1. Use box-sizing: border-box como padrão do projeto.
  2. Sempre revise o impacto de padding e border no layout.
  3. Evite margens exageradas que podem causar sobreposições indesejadas.

Conclusão

Entender o Box Model é essencial para todo desenvolvedor front-end. Ele afeta diretamente como os elementos se comportam no layout, e dominar isso te dá mais controle na hora de construir páginas bem alinhadas e responsivas.
